You are mapping benchmark task evidence to a fixed cognitive ability vocabulary.

For each ability tag, decide how strongly the provided task samples require that
ability. Use only the supplied evidence. Do not infer from benchmark names,
leaderboard scores, model names, or external reputation.

Calibration rules:
- Tag broad cognitive operations, not domains, topics, answer formats, or
  benchmark identity.
- Different-looking tasks should receive similar levels for an ability when the
  same operation is repeatedly required.
- Do not set an ability to absent merely because it is not the benchmark's
  headline skill; use weak or medium when there is direct but secondary
  evidence.
- Avoid uniformly high or uniformly low vectors. Use the full level scale when
  evidence supports real contrasts.
- If the evidence is simple or knowledge-heavy, still mark shared operations
  such as comprehension, factual recall, rule use, or basic multi-step
  reasoning when they are directly visible.

Return JSON with this shape:
{
  "ability_levels": {
    "<tag_id>": "absent|weak|medium|strong|dominant"
  },
  "ability_evidence": {
    "<tag_id>": "brief evidence from the samples"
  },
  "chunk_summary": "one short summary of the task requirements"
}

Level meanings:
- absent: not required by the chunk
- weak: occasional or peripheral
- medium: clearly present but not central
- strong: central for many items
- dominant: the primary ability needed for almost every item
